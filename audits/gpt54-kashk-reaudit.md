# Kashk-e Bademjan re-audit — remaining issues only

- **Most of the newly added sections still do not render on the live page.** Missing from the visible DOM/UI: **dietary badges, allergen warning, cultural background, flavor profile, equipment, common mistakes, regional variations, make ahead, occasions, and cost estimate**. There is also an **empty wrapper under the byline/meta row**, which looks like the intended render area for some of these blocks.
- **Nutrition is still incomplete.** The on-page nutrition card renders, but only shows **Calories**. JSON-LD `nutrition` also only contains `calories` — **no `proteinContent`, `fatContent`, or `carbohydrateContent`**.
- **SEO meta still looks like fallback content, not dedicated SEO fields.** `<title>` is just `Kashk-e Bademjan (...) - Zaffaron`, and the description / OG / Twitter descriptions are a **truncated excerpt of the body copy**, not a custom SEO description.
- **Visual/layout:** no major breakpoint failure seen on desktop or at ~390px mobile width, and **Jump to Recipe works**. The main remaining UI issue is that the page still feels **visually thin/incomplete** because the promised new sections are absent; the nutrition card also looks **sparse** because only one metric renders.
- **Gap vs industry best (NYT Cooking / Serious Eats):** still missing the high-value, scan-friendly support blocks that help cooks before they start — especially **equipment, pitfalls, make-ahead, variations, cultural context, and clear allergen/dietary callouts**.
- **Accessibility:** the rendered new blocks use acceptable heading structure/contrast, but the missing **allergen warning** and **dietary badges** mean important scan-first information is still not exposed near the top in a clear semantic/visual way.
